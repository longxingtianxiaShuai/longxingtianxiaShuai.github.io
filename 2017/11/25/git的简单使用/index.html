git的简单使用

前言
如果你想把代码保存到码云上,有没有合适的图形化界面管理工具,就要学一下git的命令行用法了。好了废话不多说 ，先说几个基本概念

基本概念

  • 1 本地文件被称为 工作区
  • 2 git add 是告诉Git 把哪些文件添加到暂存区（仓库）
  • 3 git commit 实际上就是把暂存区的所有内容提交到当前分支
  • 4 git push -u origin master 把文件提交到远程仓库

一些常用指令的使用场景

touch readme.txt
git status 查看当前仓库的状态
git diff readme.text 查看修改了哪些内容

管理修改

HEAD指向的版本就是当前版本，因此，Git允许我们在版本的历史之间穿梭，使用命令git reset –hard commit_id。

穿梭前，用git log可以查看提交历史，以便确定要回退到哪个版本。

要重返未来（比如HEAD 现在指向100，要跳转到200），用git reflog查看命令历史，以便确定要回到未来的哪个版本。

提交修改的形式
  • 1第一次修改 -> git add -> 第二次修改 -> git add -> git commit
  • 2 第二次修改后执行一次-> git add -> git commit

撤销修改

分三种情况

  • 1 还没有执行 git add readme.txt 通过执行 git checkout – readme.txt 撤销修改
  • 2 执行了git add readme.txt 分两步
  • 1 执行 git reset HEAD readme.txt 到工作区
  • 2 执行git checkout – readme.txt 撤销修改
  • 3 执行了git commit “提交了readme.txt”
  • 1 git reset –hard HEAD^ 会退到上一个版本
  • 2 git reset –hard HEAD^^回退到上上个版本
  • 3 git reset –hard HEAD~100 向上回退100个版本
  • 4 git reset –hard 3628164 回退到指定版本 可以通过 git reflog 查看要回到哪个版本

文件删除

  • 1 执行 rm readme.txt 删除工作区的文件
    此时有两种情况供你选择
  • 1 删错了 执行 git checkout readme.txt 从暂存区 恢复
  • 2 确定要删除 执行 git rm readme.txt 并且执行 git commit -m “提交修改”
  • 3 git rm 后执行git checkout readme.txt 就无效了 如果readme.txt 执行过commit 可以通过git reset –hard 3628164会到指定版本 之后执行 git push -u origin master 提交到远程仓库
  • 4 命令git rm用于删除一个文件。如果一个文件已经被提交到版本库，那么你永远不用担心误删，但是要小心，你只能恢复文件到最新版本，你会丢失最近一次提交后你修改的内容

远程仓库

  • 1 在本地创建文件夹 test -> cd test -> git init
  • 2 关联远程仓库 git remote add origin 仓库地址
  • 3 git pull origin master
  • 4 修改文件
  • 5 git push -u origin master
    tip
    第一次推送master分支是 加上 -u 参数,git不但会把本地的master分支和远程的master分支关联起来.在以后的推送或拉取的时候可以简化命令

创建和合并分支

  • 查看分支 git branch
  • 切换分支 git checkout 分支名称
  • 创建分支 git branch dev
  • 创建+切换分支 git checkout -b 分支名称
  • 合并某分支(dev)到当前分支(master) {
    git checkout master
    git merge dev
    }

—- 以上是还没有添加到远程仓库的操作 —

git config —-global user.name “why”
